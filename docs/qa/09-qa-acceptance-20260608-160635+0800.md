## 2. P0 / P1 / P2 列表

### P0 — 阻断上线

1. **生产站点不可访问**
   - 证据：`https://image-editor.co`、`https://www.image-editor.co` 返回 Cloudflare 522。
   - 影响：用户无法访问站点。
   - 复现：`curl -I -L https://image-editor.co`

2. **生产 API 不可访问**
   - 证据：`https://api.image-editor.co` 返回 Cloudflare 522。
   - 影响：session、上传、编辑、copy rewrite、用量查询均无法使用。
   - 复现：`curl -I -L https://api.image-editor.co`

3. **D1 数据库无表**
   - 证据：`wrangler d1 list` 显示 `rsp-db` 的 `num_tables = 0`。
   - 影响：`session/init`、usage、edit task、entitlement 等依赖 DB 的接口无法正常运行。
   - 修复方向：执行 remote migration，并验证表结构。

4. **AI 编辑主链路未实现**
   - 证据：`workers/ai.ts` 中 `cf-workers-ai` 分支直接抛 TODO。
   - 影响：`enhance` / `remove-bg` / `restyle` 无法真实产出结果。
   - 修复方向：确认 AI provider，填入真实模型与密钥，完成端到端上传编辑导出测试。

5. **R2 输出 URL 仍含占位逻辑**
   - 证据：`workers/entry.ts` 使用 `https://placeholder.r2.dev/...`。
   - 影响：导出 URL 不能作为生产可访问资产链接。
   - 修复方向：使用 R2 public domain、signed URL 或 Worker proxy。

### P1 — 修复后需要复测

1. **Copy rewrite 为占位返回**
   - 证据：`workers/entry.ts` 的 `/api/v1/copy/rewrite` 返回 `[style] text` 形式占位版本。
   - 影响：文案改写功能不是实际 AI 能力。
   - 建议：接真实 LLM，或上线前在 UI 中明确标 beta/mock，避免误导。

2. **生产域名口径不一致**
   - 证据：
     - 前端 API 默认：`https://api.image-editor.co`
     - 当前线上候选：`image-editor.co`
     - metadata/sitemap/robots：`https://rsp-ai-editor.com`
   - 影响：SEO、canonical、OG URL、sitemap 可能指向错误域名。
   - 建议：冻结最终域名后统一 `metadataBase`、OG、robots、sitemap、API URL。

3. **缺 PM / SEO / 合规 Gate 明确 GO 文件**
   - 影响：上线责任边界不清。
   - 建议：补一份 gate handoff，列明是否允许上线、允许哪些功能上线、哪些功能需隐藏或降级。

4. **Lint warnings 未清理**
